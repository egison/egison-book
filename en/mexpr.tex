\chapter{Handling Mathematical Expression Data}\label{mexpr}

This chapter introduces Egison's features for handling symbolic computation.

\section{Internal Representation of Mathematical Expression Data}\label{mexpr-expr}

Mathematical expressions are directly implemented as a built-in data type within the Egison interpreter using Haskell data types.
This section explains the internal data structure that represents mathematical expression data.

Mathematical expression data is represented as a fraction that takes polynomials in sum-of-products normal form for both the numerator and denominator.
Therefore, for example, the result of adding two fractions whose denominators are polynomials is combined into a single fraction over the common denominator.
\begin{lstlisting}[language=egison,numbers=none]
declare symbol x, y, a, b, c

a / (b + 1) + x / (y + 1)
-- (b x + a y + x + a) / (b y + y + b + 1)
\end{lstlisting}
A polynomial is represented as data consisting of a list of terms.
A term is represented as data consisting of a coefficient and an association list of factors and exponents.
For example, the term \lstinline{3 * x^2 * sqrt 2} has \lstinline{3} as the coefficient and \lstinline{[(x, 2), (sqrt 2, 1)]} as the association list.
Exponents can also be negative integers.
That is, polynomials in Egison are Laurent polynomials.
A fraction whose denominator consists of a single term is not kept as a fraction; it is folded into the terms using negative exponents.
\begin{lstlisting}[language=egison,numbers=none]
a / b + x / y -- x y^-1 + a b^-1
\end{lstlisting}

Besides symbols, factors can be the symbolic function applications introduced in Section~\ref{cas-symbol2}, the quoted expressions introduced in Section~\ref{mexpr-backquote}, and the function symbols explained in Chapter~\ref{funcsym}.
A symbol consists of its name string and a list of indices.
Indices are explained in Chapter~\ref{tensor}, which covers tensors.

This internal structure can be decomposed by pattern matching with the \lstinline{mathValue} matcher defined in the Egison library.
For example, a polynomial can be decomposed into its list of terms.
\begin{lstlisting}[language=egison,numbers=none]
match (x - sqrt 2) as mathValue with
  | poly $ts -> ts
-- [x, - 'sqrt 2]
\end{lstlisting}
Patterns for mathematical expressions are introduced in detail in Section~\ref{mexpr-pattern}.
Furthermore, the same mathematical expression can be represented by several different internal structures (for example, a flat polynomial and a nested polynomial), and the structure can be selected with type annotations.
This is explained in Chapter~\ref{casdetail}.

Numbers are a complex data type.
Up to natural numbers, they can be elegantly defined as algebraic data types.
However, when considering subtraction, the inverse of addition over natural numbers, the concept of negative numbers emerges.
Furthermore, considering the inverse of multiplication, which is obtained by repeated addition, leads to the concept of rational numbers.
Moreover, considering the inverse of exponentiation, which is obtained by repeated multiplication, leads to the concepts of square roots, imaginary numbers, and algebraic numbers that include them.

\section{Controlling Expression Expansion with Backquotes (\texttt{`})}\label{mexpr-backquote}

As mentioned in Section~\ref{cas-symbol}, Egison automatically expands mathematical expressions into sum-of-products normal form, but this expansion can be controlled using backquotes (\lstinline{`}).
An expression following a backquote is treated like a single symbol.
\begin{lstlisting}[language=egison,numbers=none,escapechar=']
`(x + 1)^2
--  `(x + 1)^2

`(x + 1) + `(x + 1)
--  2 * `(x + 1)

`(x + 1) + (x + 1)
-- x + `(x + 1) + 1
\end{lstlisting}

Backquotes are useful when we do not want expressions to be expanded.
Reduction of polynomial fractions itself is performed automatically by polynomial GCD, as mentioned in Section~\ref{cas-simplify}, but both the input and the result are expanded into sum-of-products normal form.
\begin{lstlisting}[language=egison,numbers=none,escapechar=']
(x + 1)^2 / (x + 1) -- x + 1
(x^2 - y^2) / (x - y) -- y + x
\end{lstlisting}
If the factored form is preserved using backquotes, the expression is reduced without going through expansion, and the result is also obtained in factored form.
\begin{lstlisting}[language=egison,numbers=none,escapechar=']
`(x + y)^2 / `(x + y) -- `(y + x)
`(x - y) * `(x + y) / `(x - y) -- `(y + x)
\end{lstlisting}

An expression quoted with a backquote is treated as a factor, on the same footing as a symbol, within Egison's internal mathematical expression data.
This is why \lstinline{`(x + 1)^2} above is kept as a power of a single factor without being expanded.

\section{Controlling Function Application with Single Quotes (\texttt{'})}\label{mexpr-squote}

For applications of defined functions, we sometimes want to return a symbolic result without evaluating the function application.
For example, we want the \lstinline{sqrt} function to behave as follows, as we saw in Section~\ref{cas-symbol2}.
\begin{lstlisting}[language=egison,numbers=none]
declare symbol x, a

sqrt 4 -- 2
sqrt x -- 'sqrt x
\end{lstlisting}
To realize this behavior, Egison provides single quotes (\lstinline{'}) as a built-in syntax.
A single quote is followed by a function, and even if that function is defined, the application is returned as a symbolic function application without being evaluated.
\begin{lstlisting}[language=egison,numbers=none]
def f (x: MathValue) : MathValue := x + 1

f a -- a + 1
'f a -- 'f a
\end{lstlisting}
Symbolic function applications are also displayed with a leading single quote in the output.
Single quotes are used in the bodies of the \lstinline{declare apply} statements introduced in Section~\ref{casdetail-mathfunc} --- for example in the definitions of \lstinline{sqrt} and the trigonometric functions \lstinline{sin} and \lstinline{cos} --- to return symbolic values for inputs that cannot be simplified further.

\section{Declaring Local Symbols with \texttt{withSymbols} Expressions}\label{mexpr-withsym}

Symbols declared with the \lstinline{declare symbol} statement are global.
For cases where symbols are needed only temporarily, Egison provides the \lstinline{withSymbols} expression, a syntax for declaring local symbols.
The \lstinline{withSymbols} expression takes a list of variables as its first argument, and these variables are treated as symbols during the evaluation of the expression in the second argument.
Even if a variable is already defined, it is treated as a symbol inside the \lstinline{withSymbols} expression.
\begin{lstlisting}[language=egison,numbers=none]
def k : Integer := 10

k -- 10
withSymbols [k] k -- k
k + withSymbols [k] k -- k + 10
\end{lstlisting}

Local symbols declared with a \lstinline{withSymbols} expression are processed as symbols independent of the global symbols declared with the \lstinline{declare symbol} statement.
The two \lstinline{j}s below are displayed with the same name, but since they are different symbols, they are not combined into \lstinline{2 * j}.
\begin{lstlisting}[language=egison,numbers=none]
declare symbol j

j + withSymbols [j] j -- j + j
\end{lstlisting}

\section{Pattern Matching on Mathematical Expression Data}\label{mexpr-pattern}

This section introduces the patterns provided for mathematical expression data.
The matcher \lstinline{mathValue} for mathematical expressions is implemented in the Egison library.
The definition of \lstinline{mathValue} can be found in \texttt{lib/math/expression.egi}.
In addition, several syntactic sugars are built in for writing patterns for mathematical expressions in a form closer to mathematical notation.
This section also introduces these syntactic sugars.
Pattern matching on mathematical expressions is demonstrated in Section~\ref{cas-derivative}.
Please refer to the program in Section~\ref{cas-derivative} for concrete examples of using these patterns.

\subsection{Pattern Matching on Factors}

The pattern constructors for factors are \lstinline{symbol}, \lstinline{apply1} through \lstinline{apply4}, \lstinline{quote}, and \lstinline{func}.
\lstinline{symbol} pattern-matches symbols; \lstinline{apply1} through \lstinline{apply4} pattern-match the symbolic function applications introduced in Section~\ref{cas-symbol2}; \lstinline{quote} pattern-matches backquoted expressions introduced in Section~\ref{mexpr-backquote}; and \lstinline{func} pattern-matches objects called function symbols, explained in Chapter~\ref{funcsym}.

The first argument of the \lstinline{symbol} pattern constructor pattern-matches the name string of the symbol, and the second argument pattern-matches the list of indices.
Indices are covered again in Chapter~\ref{tensor}, which explains tensors.
\begin{lstlisting}[language=egison,numbers=none]
match x as mathValue with
  | symbol $s _ -> s
-- "x"
\end{lstlisting}
Although symbols can be pattern-matched using the \lstinline{symbol} pattern constructor, in practice, value patterns are more commonly used.
\begin{lstlisting}[language=egison,numbers=none]
match x as mathValue with
  | #x -> "Matched"
  | _ -> "Not matched"
-- "Matched"
\end{lstlisting}

For symbolic function applications, the pattern constructors \lstinline{apply1} through \lstinline{apply4} are used according to the number of arguments.
The first argument pattern-matches the function itself, and the remaining arguments pattern-match the expressions of the respective arguments.
For the function itself, value patterns such as \lstinline{#sqrt} are commonly used.
\begin{lstlisting}[language=egison,numbers=none]
match (sqrt x) as mathValue with
  | apply1 #sqrt $g -> g
-- x
\end{lstlisting}

The \lstinline{quote} pattern constructor takes a pattern for mathematical expressions as its argument.
This pattern is matched against the expression inside the quote using the \lstinline{mathValue} matcher.
\begin{lstlisting}[language=egison,numbers=none,escapechar=']
match `(x + 1) as mathValue with
  | quote $e -> e
-- x + 1
\end{lstlisting}


\subsection{Pattern Matching on Terms}

The \lstinline{term} pattern constructor is used for pattern matching on terms.
The first argument of the \lstinline{term} pattern constructor pattern-matches the coefficient, and the second argument pattern-matches the association list of factors and exponents.
The assocMultiset matcher introduced in Section~\ref{matcher-assoc} is used for pattern matching the association list of factors.
\begin{lstlisting}[language=egison,numbers=none]
matchAll 3 * x^2 * y as mathValue with
  | term $a (($x, $n) :: $xs) -> (a, x, n, xs)
-- [(3, x, 2, [(y, 1)]), (3, y, 1, [(x, 2)])]
\end{lstlisting}

The \lstinline{term} pattern constructor has syntactic sugar for writing patterns in a form closer to mathematical notation.
In the first pattern below, \lstinline{$x} matches a factor together with its power, such as \lstinline{x^2}, and \lstinline{$xs} matches the product of the remaining factors.
As in the second pattern, a pattern for the exponent can also be written following \lstinline{^}.
\begin{lstlisting}[language=egison,numbers=none]
matchAll 3 * x^2 * y as mathValue with
  | $a * $x * $xs -> (a, x, xs)
-- [(3, x^2, y), (3, y, x^2)]

matchAll 3 * x^2 * y as mathValue with
  | $a * $x^#2 * $xs -> (a, x, xs)
-- [(3, x, y)]
\end{lstlisting}

\subsection{Pattern Matching on Polynomials}

The \lstinline{poly} pattern constructor is used for pattern matching on polynomials.
The \lstinline{poly} pattern constructor takes a pattern that matches a list of terms as its argument.
\begin{lstlisting}[language=egison,numbers=none]
matchAll x + y + 1 as mathValue with
  | poly ($x :: $xs) -> (x, xs)
-- [(y, [x, 1]), (x, [y, 1]), (1, [y, x])]
\end{lstlisting}

The \lstinline{poly} pattern constructor also has syntactic sugar for writing patterns in a form closer to mathematical notation.
\begin{lstlisting}[language=egison,numbers=none]
matchAll x + y + 1 as mathValue with
  | $x + $xs -> (x, xs)
-- [(y, x + 1), (x, y + 1), (1, y + x)]
\end{lstlisting}

\subsection{Pattern Matching on Rational Expressions}

The \lstinline{frac} pattern constructor is used for pattern matching on rational expressions.
The first argument of the \lstinline{frac} pattern constructor pattern-matches the numerator polynomial, and the second argument pattern-matches the denominator polynomial.
\begin{lstlisting}[language=egison,numbers=none]
matchAll (a + b) / (c + 1) as mathValue with
  | frac $x $y -> (x, y)
-- [(b + a, c + 1)]
\end{lstlisting}
Note that, as mentioned in Section~\ref{mexpr-expr}, a fraction whose denominator consists of a single term is represented as a term with negative exponents, and hence does not match the \lstinline{frac} pattern.

The \lstinline{frac} pattern constructor also has syntactic sugar for writing patterns in a form closer to mathematical notation.
\begin{lstlisting}[language=egison,numbers=none]
matchAll (a + b) / (c + 1) as mathValue with
  | $x / $y -> (x, y)
-- [(b + a, c + 1)]
\end{lstlisting}
